# Acceptable Use Policy

**Document ID:** POL-02
**Version:** 1.0
**Effective Date:** 2026-01-15
**Approved By:** CISO
**Review Cycle:** Annual

---

## 1. Purpose

Define acceptable use of NexPay information technology resources to protect the organization from legal liability, security incidents, and productivity loss while respecting employee privacy.

---

## 2. Scope

All employees, contractors, and third parties accessing NexPay IT resources including: workstations, email, internet, cloud services, mobile devices, and corporate networks.

---

## 3. Policy Requirements

### 3.1 General Use
- Company resources are provided for business purposes. Incidental personal use is permitted so long as it does not violate policy, consume excessive resources, or create security risk.
- Users must not engage in activities that violate law, regulation, or NexPay policies.
- Users must not access, store, or transmit illegal, offensive, discriminatory, or harassing content.

### 3.2 Email & Communications
- Corporate email must not be used for personal financial accounts, illegal activities, or non-business solicitation.
- Users must not send threatening, harassing, or discriminatory messages.
- Phishing awareness: Users must report suspicious emails via the Phish Alert button.
- Auto-forwarding of corporate email to external addresses is prohibited.

### 3.3 Internet & Cloud Services
- Users must not bypass security controls (VPN, proxy, firewall) to access restricted websites.
- Use of personal cloud storage (Google Drive, Dropbox, iCloud) for company data is prohibited unless explicitly approved.
- File sharing services must be approved and must comply with data classification requirements.
- Cryptocurrency mining, P2P file sharing, and torrenting are prohibited.

### 3.4 Software Installation
- Only approved, licensed software may be installed on corporate devices.
- All software installations require IT approval. Self-admin rights for developers require explicit exemption.
- Open-source software must be reviewed for license compliance and security posture.

### 3.5 Personal Devices (BYOD)
- BYOD devices must have a screen lock, full-disk encryption, and the ability to remotely wipe corporate data.
- Company data must not be stored locally on personal devices unless approved (use virtual desktop or cloud apps).
- NexPay reserves the right to wipe corporate data from personal devices upon termination or security incident.

### 3.6 Monitoring & Privacy
- NexPay monitors IT resources for security and compliance purposes. Users have no expectation of privacy on company systems.
- Monitoring includes: network traffic, email content (automated scanning), web browsing, application usage, and file access logs.
- Monitoring is conducted in accordance with applicable privacy laws and employee consultation requirements.

### 3.7 Prohibited Activities
- Unauthorized access, hacking, scanning, or penetration testing
- Sharing credentials or using another user's account
- Introduction of malicious software
- Deliberate circumvention of security controls
- Storage of sensitive data on unsanctioned devices or services

---

## 4. Enforcement

Violations result in disciplinary action up to termination. Managers must report known violations to HR and Security.
